for operation . 
DESCRIPTION  
Winegard has identified an issue with some Trav’ler Pro products when the auto- stow feature is 
enabled. In a specific scenario, when  the auto- stow feature is active, the firmware may occasionally 
encounter difficulty in recognizing the stowed position. This bulletin supersedes bulletin (607T) now 
that Winegard has released the firmware update for a complete resolution.  
ISSUE  
This situation may lead to an occasional unintended stow command when there is a power 
interruption.  This may cause the antenna to lift, rotate,  and return to the stowed position.  
UNITS AFFECTED  
Please refer to the attached population list for affected units and VINs.

Winegard has now developed a firmware update that needs to be installed. (See the attached links to 
view the tutorial videos on how to complete this process ).

Clariﬁcation from Winegard on  Update Issues 
1. Currently the IDU’s WiFi scan will not ﬁnd any SSID that contains an apostrophe. So a 
hotspot named Zack’s iPhone or Bryan’s iPhone would not be listed.   (Not ideal,  we realize. ) 
2. The app’s progress bar is simply an animation and is not directly proportional to the IDU or 
ODU progress.   The real progress is displayed on the IDU screen.  
3. When you see the update start off with an ODU update, that means the antenna (ODU) and 
IDU control box have Firmware mismatched from the Newmar plant.   This happens when an 
IDU on older ﬁrmware gets installed with an antenna on newer ﬁrmware.   When the tech 
sees the TSB update process begin with an ODU download the tech is witnessing i s the IDU 
rolling back the ODU ﬁrmware to an older but compatible version of ﬁrmware.   This roll back 
does not require i nternet access.   The ODU ﬁrmware and installer reside permanently in the 
IDU.  This roll back feature exists should an antenna need replacement years in the future, 
the original IDU can ensure the ODU antenna is rolled back to compatible 
ﬁrmware.   (Antenna and IDU need compatible ﬁrmware to ensure accurate app control ).  As 
of June 2024, the factory keeps the IDU and antenna as a matching set.  
4. When the QR code does not allow app pairing, it’s because someone has already paired a 
device to that IDU.   If that someone forgets to press “Remove Device from RV” in the app, it 
will prevent a second person from pairing to the IDU.   Winegard uses a bonde d Bluetooth 
pairing with secure key.   That’s how we prevent unauthorized persons from pairing to a 
customer’s Travler Pro IDU.   If a tech runs into this issue, pressing and holding the reset 
button on the back of IDU until the IDU display goes dark is the only way to purge the pairing 
key.  Once the pairing key is purged from IDU, a new user can then scan the QR and pair to 
the IDU. 
5. Any non -mismatched IDU / ODU will upgrade straight to the newest ﬁrmware 
versions.   There are no sequential update hops.   (The explanation in item 3 above clariﬁes 
things.) 
6. Once the IDU update is complete (download, installation, reboot), internet access is no 
longer required.   The ODU update is ready and waiting inside the updated IDU.   The tech just 
needs to start the ODU update via app.  
7. Both IDU and ODU updates are needed to satisfy the TSB and ensure proper Auto Stow 
functionality.  
8. Cloud icon on IDU must illuminate.   An illuminated Cloud icon indicates the IDU has 
internet access to Winegard’s upgrade server.   The “!” exclamation point inside the Cloud 
icon indicates there is an update job available for download.   I have seen some WiFi 
networks prevent connectivity to the Winegard upgrade server, so the cloud icon ne ver 
illuminates.   Newmar’s Guest network is an example.   That network allows general i nternet 
access, but blocks lots of things.   So, keep in mind that a dealership may ne ver get the 
cloud to illuminate on an IDU.   If this is the case, it’s something on their network (an IT rule 
or restriction) preventing the connection.  
9. We don’t allow the IDU to connect to an open unsecured network.   We don’t want our 
ﬁrmware payload transmitted across an unsecured network.  
10. The ODU update is slower than the IDU’s.   That is normal behavior.   The ODU update is 
transmitted up the 6 pin control cable that connects the IDU to the ODU.   We have to run a 
slower baud rate to ensure data transmission accuracy.  
11. There were concerns expressed about ODU update starting ﬁrst.   Number 3 explains the 
how and why.  
12. The only time a Travler Pro antenna autostow event will begin is when these 4 criteria are 
met:  Travler Pro IDU has AC power, IDU displays “Not Stowed ,” the coach engine is running, 
and the park brake is released.  
13. The tech (or owner) can also see Firmware versions from screen of IDU.   I ﬁnd that easier 
than using app.
